Protocol Documentation
Communication Protocol
This application uses the proprietary UART-TTL protocol (internally called "Protocol 000"), which is the native communication method for JK-BMS devices via the GPS/UART port. This is NOT the Modbus RS-485 protocol that requires an external adapter.

Frame Structure:

text
Byte 0-3:   Header    0x55 0xAA 0xEB 0x90
Byte 4:     Type      0x01 (configuration) or 0x02 (telemetry)
Byte 5-299: Payload   Data (little-endian)
Byte 300:   Device ID 0x00=Master, 0x01-0x03=Slave
Byte 301-307: Footer  (frame padding)
Total:      308 bytes (fixed length)
Frame Types:

Type 0x01: Configuration/settings data (ignored by this application)

Type 0x02: Real-time telemetry data (processed by this application)

Not Available
The following parameters appear in the mobile app but are not available via the UART-TTL protocol:

Bulk/Absorption/Float charging mode (internal BMS logic only)

Cycle count (offset found but values inconsistent)

Troubleshooting
No Data or Missing Batteries
Check BMS Addresses: Each BMS must have a unique address (0x00-0x03) configured in the mobile app

Verify Serial Port: Confirm the correct port path (/dev/ttyUSB0, COM3, etc.)

Check Wiring: Ensure TX/RX are cross-connected (BMS TX → Converter RX, BMS RX → Converter TX)

Communication Protocol: The BMS UART port must be set to "Protocol 000" in the mobile app settings

Only Master Connected: Only the MASTER BMS should be physically connected to the converter; slaves communicate through the RS-485 bus


SparkFun Converter Issues

Port Detection: Check with ls /dev/tty* (Linux) or Device Manager (Windows)

Permissions: On Linux, add user to dialout group: sudo usermod -a -G dialout $USER

Baud Rate: Must be 115200 bps (configured in the code)

Incorrect Values
Cell Voltages: Only values between 3.0V and 3.8V are accepted (LiFePO4 range)

Temperature Readings: If temperatures show 25.7°C on all sensors, the sensor may be disconnected

Current Direction: Positive = charging, Negative = discharging

Missing Cells: The BMS frame supports 32 cells, but only 15 are active (remaining show 0V)

Known Limitations
Hardware Specific: Validated only on JK-PB1A16S-10P with firmware V19.x

15 Cells Maximum: Application reads 15 cells (BMS supports up to 16)

No Modbus Support: This is UART-TTL protocol only, not compatible with the RS-485 Modbus adapter

No Charging Mode: Bulk/Absorption/Float states are not exposed via UART-TTL protocol
